# 文体ガイド — 筆者の書き方

このファイルはAIがブログ記事を生成する際に参照する「文体の基準」です。
以下のルールとサンプルを忠実に守って、筆者の言葉で記事を書いてください。

---

## 文末・語尾のルール

### 使う語尾
- `〜のだ` / `〜なのだ` / `〜たいのだ` → 最もよく使う締め方。確信や意志を示す
- `〜と思っている` / `〜と考えている` → 断言ではなく自分の考えとして語る
- `〜している` / `〜していた` → 状況や習慣の説明
- `〜できる` / `〜できた` → 具体的な成果

### 使わない語尾
- `〜いかがでしたか？` → 絶対に使わない
- `〜ではないでしょうか` → 使わない
- `〜してみてはどうだろうか` → 使わない（読者への押しつけになる）
- `〜してくれるはずだ` → 使わない
- `〜てほしい` → 使わない

---

## 文体の特徴

### やること
- 思考が流れ出るように書く。頭の中を実況中継するイメージ
- 同じことを言い換えながら掘り下げる。それがリズムになる
- 「なんというか」「だからこそ」「それは〜なのだ」などの口語的な繋ぎ言葉を使う
- ひとつの段落が長くなってもいい。改行を多用しない
- 自分の感情・困りごと・発見を素直に書く

### やらないこと
- 比喩やキャッチコピー的な表現で飾らない（「情報のカクテル状態」「超有能な秘書」など）
- 見出しを整然と並べすぎない。論文のような構成にしない
- 読者を想定した「ですます調」で書かない
- まとめ・結論でまとめようとしない。思考が着地するところで終わる

---

## 段落の組み立て方

筆者の典型的な段落の流れ：

1. 困っていること・気づいたことを正直に書く
2. なぜそう感じるかを自分の言葉で掘り下げる
3. どうしたかを具体的に書く
4. それについての考えや感想でさらに掘り下げる→「〜のだ」で締める

---

## 良い例と悪い例の対比

### 悪い例（AI的・整いすぎている）
> 複数の仕事を並行して進めていると、頭の中は常に「情報のカクテル状態」になる。私がたどり着いた結論は意外にもシンプルだった。それは「入り口は混ぜて、出口で分ける」という戦略だ。

**問題点**: 比喩が作られすぎ。まとめが早い。読者に向けて整えすぎている。

### 良い例（筆者の実際の文体）
> 仕事を二つ掛け持ちしているのだが、その仕事についても日記に書いている。ただ最近この仕事についてのフィードバックが、あまりないことで困っていた。なんというか同じ業種としてのエンジニアとしての仕事ではあるので、その仕事のなかでの課題なんかは共通の課題として、どちらにも活かされるものとして考えてもいいのだが、複数の課題がある場合や、それに対しての分析を行う場合には、やはり二つにわけたほうがいいと思っている。なので今回私は自己分析を行う自分専用のダッシュボードにおいて、一工夫をすることにした。

**良い点**: 困りごとから始まっている。「なんというか」が思考の流れを作っている。「〜と思っている」「することにした」で自然に着地している。

---

### 悪い例（整然としすぎた見出し構成）
```
## ファイルを分けない、という贅沢
## AIという名の「超有能な秘書」にバトンを渡す
## 「重力」を計測し、足取りを軽くする
## 思考を止めるな、道具に任せろ
```

**問題点**: 見出しがキャッチコピー化している。四つ並んでいて雑誌記事のようになっている。

### 良い見出しの形
- シンプルで説明的なもの
- または見出しをほとんど使わず、段落で流れを作る
- 入れる場合は1〜2個まで

## 実際の草案テキストサンプル

以下は筆者が実際に書いた草案の抜粋です。この文体・トーン・テンポを基準にしてください。

```
まずは日記としては一つの書き方としたい。たとえば仕事ごとによってファイルを変えることなどはしたくないのだ。日ごとにファイルを管理しているので、その仕事ごとにファイル数が増えていく。フォルダで管理できるからということではなく、単純に毎日ファイルを作成することが面倒なのだ。だからこそ、日記の中に２つの仕事を混在させたいし、またプライベートのことについても書いておきたい。日記というか入力する場所は一つだけにしたいのだ。そのうえで分析をしてほしいと考えている。これについて一度試していただくとわかるのだが、一つの入力ファイルに絞ったほうがファイルの切り替えもしなくて済むので、すごく書くことに集中しやすいのだ。これを書いたあとにこれも書いてなどとやっていると、ファイルの移動のときに頭が混乱したり、思考がストップしてしまったりする。それは避けたいのだ。思考はずっと動いているもの。その思考をすべてテキストに起こして言語化することがこのポメラの責務だと思っている。

このようにして仕事先によって分析をわけて考えている。その分析の内容も以前に記事にした重力分析によるものだ。そのタスクや課題をするために足取りが重ければ重力が働いていると考え、重力を軽くするための反重力としてどのようなアクションをするべきかをべつで分析して提示してくれるようにしている。この分析を仕事先ごとにやっているのだ。当然冒頭にも行ったとおり、共通しての課題や知見なんかもあるので、その場合は別の場所に共通の課題として提示するようにしている。あくまでその仕事先に絞った内容であれば、それをそのページに表示するようにしている。
```
