# ポメラ日記 to ナレッジグラフ：運用ガイド

日々の運用フロー（日記執筆から分析まで）の手順書です。

## 1. ポメラでの執筆（毎日）

日記を自由に書きつつ、以下の「魔法のタグ」を使ってLLMにヒントを与えます。

### 📝 書き方のコツ
- **自由記述**: 感情や出来事をありのままに書く。
- **予定タグ**: `予定::YYYY/MM/DD イベント名`
    - AIに「この日は何があるか」を伝えます。AIはこれを解釈し、関連するイベントとして認識します。
- **目標タグ**: `目標::ゴール名`
    - AIに「今何を目指しているか」の文脈を共有します。これによってアドバイスの精度が上がります。
    - ※注意: タグの内容がそのままノード名になるわけではなく、AIが文脈に合わせて整理・統合することがあります。

**例:**
```text
今日は仕事でミスをして落ち込んだ。でも同僚が励ましてくれた。
帰りに美味しいカレーを食べて復活。

予定::2026/02/20 オンライン飲み会
目標::毎朝7時に起きる
```

---

## 2. PCへの取り込み（毎日）

1.  ポメラをPCに接続（またはQRコード転送）。
2.  日記ファイルを所定のフォルダ（`Documents/ポメラtoナレッジグラフ`）に保存します。
    - ファイル名例: `daily_diary.txt`（毎回同じ名前に上書きしても、日付を変えてもOK）

---

## 3. 分析の実行（毎日）

ターミナルを開き、以下のコマンドを実行します。

```bash
cd ~/Documents/ポメラtoナレッジグラフ
python3 llm_graph_builder.py daily_diary.txt
```

### 自動で行われること
1.  **グラフ抽出**: 今日の日記からノードとエッジを取り出します。
2.  **記憶の統合**: 過去のデータ（`master_graph.json`）と合体させます。
3.  **分析**: 過去の目標と今日の行動を比較し、アドバイスを生成します。
4.  **可視化更新**: `index.html` のデータを最新化します。

---

## 4. 結果の確認（毎日）

### 📊 レポートを見る (`daily_report.md`)
生成された Markdown ファイルを開きます。
- **Plan vs Actual**: 目標に対してどうだったか？
- **Advice**: 明日具体的に何をすべきか？

### 🕸️ グラフを見る (`index.html`)
ブラウザで `index.html` を開きます（またはリロード）。
- **自分（中心）**: 今日の出来事がつながっています。
- **未来（周辺）**: `予定::` で書いたイベントが、未来の日付として配置されています。

---

## 💡 ヒント

*   **過去の蓄積**: 毎日繰り返すことで `master_graph.json` が育ち、AIのアドバイスがより個人の文脈に沿ったものになります。
*   **データの修正**: `daily_diary.txt` を書き直して再度コマンドを実行すれば、何度でも分析をやり直せます。
